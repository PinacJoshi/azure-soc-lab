# 🛡️ Azure SOC Lab — Stage 1: Cloud Infrastructure & Key Vault Deception

A student azure subscription, cloud-native Security Operations Center (SOC) lab built inside Microsoft Azure using **Terraform**, **Microsoft Sentinel**, and **Deception Engineering**. 

Stage 1 establishes Infrastructure as Code (IaC) modularity, deploys Microsoft Sentinel SIEM, provisions bait resources (Key Vault & Storage Account), routes diagnostic telemetry, and deploys Detection as Code via custom KQL Analytic Rules.

## 🧰 Tech Stack & Prerequisites

* **Cloud Provider:** Microsoft Azure
* **Infrastructure as Code (IaC):** Terraform (`azurerm ~> 4.0`, `azuread ~> 2.47`, `random ~> 3.6`)
* **SIEM & Logs:** Microsoft Sentinel, Log Analytics Workspace (LAW), Kusto Query Language (KQL)
* **Deception Assets:** Azure Key Vault, Azure Blob Storage
* **CLI Tools:** [Azure CLI](https://learn.microsoft.com/en-us/cli/azure/install-azure-cli), [Terraform CLI](https://developer.hashicorp.com/terraform/downloads) (`>= 1.5.0`), `curl`

## ⚙️ Deployed Components

| Resource | Azure Service | Description / Technical Rationale |
| --- | --- | --- |
| `azurerm_resource_group` | Resource Group | Holds all lab resources under `rg-soc-lab-prod` for easy lifecycle management. |
| `azurerm_log_analytics_workspace` | Log Analytics Workspace | Central log repository configured with **30-day retention** for cost optimization. |
| `azurerm_log_analytics_solution` | Microsoft Sentinel | Onboards LAW to Sentinel for threat hunting and incident generation. |
| `azurerm_key_vault` + `secret` | Key Vault | Decoy vault (`kv-honeypot-*`) hosting a bait connection string secret (`db-prod-connection-string`). |
| `azurerm_storage_account` | Blob Storage | Decoy storage account (`sthoneypot*`) with a private container (`confidential-backups`). |
| `azurerm_monitor_diagnostic_setting` | Diagnostic Settings | Streams real-time `AuditEvent` logs from the Key Vault to Sentinel. |
| `azurerm_sentinel_alert_rule_scheduled` | Analytics Rule | Scheduled rule running every 5 min targeting unauthorized access attempts (MITRE T1552). |

## 🚀 Quickstart & Deployment

### 1. Authenticate with Azure CLI

```bash
az login
az account set --subscription "YOUR_AZURE_SUBSCRIPTION_ID"

```

### 2. Configure Terraform

Ensure you update `providers.tf` with your target `subscription_id` or export it as an environment variable (`ARM_SUBSCRIPTION_ID`).

### 3. Initialize & Deploy

```bash
terraform init
terraform plan
terraform apply

```

Confirm the prompt with `yes`. Deployment takes ~2–3 minutes.

## 🎯 Threat Simulation & Verification

### 1. Simulate an Unauthorized Probe

Send an unauthenticated request to the honeypot Key Vault URI endpoint:

```bash
# Replace <SUFFIX> with the random string generated by Terraform
curl -s -i "https://kv-honeypot-<SUFFIX>.vault.azure.net/secrets/db-prod-connection-string?api-version=7.4"

```

**Expected Response:** `HTTP/1.1 401 Unauthorized`

### 2. Verify Telemetry in Microsoft Sentinel

Navigate to **Microsoft Sentinel** ➔ **Logs** and run the following KQL query to inspect raw diagnostic logs:

```kusto
AzureDiagnostics
| where ResourceType == "VAULTS"
| where Resource startswith "KV-HONEYPOT"
| where ResultSignature == "Unauthorized" or httpStatusCode_d == 401 or httpStatusCode_d == 403
| project TimeGenerated, Resource, OperationName, requestUri_s, CallerIPAddress, ResultSignature, httpStatusCode_d
| order by TimeGenerated desc

```

Within **5 minutes**, Sentinel's scheduled analytics rule will process the log, flag the event, and generate an active Incident in the **Incidents** blade.
